# PLAN — Simple Visual Similarity Experiment

## Science

The experiment measures **explicit perceptual similarity** between simple visual
stimuli (colored circles). On each trial a participant sees two circles side by
side and rates how similar they look on a 5-point scale. The classic prediction
is that pairwise similarity decreases monotonically with perceptual distance
(here, hue distance around the color wheel), and that **identical pairs** are
rated maximally similar. Reaction time is recorded as a secondary measure;
intermediate (ambiguous) pairs are often the slowest to judge. Aggregating
ratings over participants yields a similarity matrix that can be visualized as a
heatmap and, in follow-up work, fed into multidimensional scaling.

## Methods

### Design

Single-factor, within-participant design. The independent variable is the
**stimulus pair**. With a set of `N` colored circles, the full set of stimulus
pairs is every **unordered** pair including identical pairs (`i <= j`), giving
`N*(N+1)/2` pairs. With `N = 6` hues this is **21 pairs**. Each participant
completes **10 random trials** drawn without replacement from the pair set, so
no pair repeats within a participant. Pairs are balanced across participants so
that, with enough participants, every pair is presented multiple times.

### Materials

The stimulus set is **6 colored circles** of fixed size, with hues evenly spaced
around the HSL color wheel (red 0°, orange 60°, yellow… — actually six hues at
0°, 60°, 120°, 180°, 240°, 300°: red, yellow, green, cyan, blue, magenta) at
full saturation and 50% lightness. Size is fixed but stored explicitly in the
stimulus manifest as a `size` field so that **size can be added as a second
varying dimension later without changing experiment code** — the runtime reads
each circle's radius from the manifest. The manifest is generated by a small
committed script (`make_stimuli.py` → `stimuli.json`) rather than hardcoded in
`experiment.py`.

### Procedure

Each trial:

1. **Fixation** — a centered black cross is shown alone for ~0.8 s.
2. **Stimulus pair** — the two circles appear simultaneously (left and right of
   center) and remain on screen until the participant responds. Responses are
   enabled at stimulus onset.
3. **Response** — participants rate similarity on a 5-point Likert scale,
   `1 = Completely Dissimilar` … `5 = Completely Similar`, by pressing the
   number keys `1`–`5` (the on-screen buttons are also clickable and labeled
   with the same digits).

For every trial we record the **exact stimulus pair** (both circle ids, hues,
hex colors, sizes), the **participant response** (rating 1–5), and the
**reaction time** (ms from stimulus onset to keypress).

A short instructions page precedes the trials, explaining the scale and the
`1`–`5` keys. No scored practice phase is included, to keep the run faithful to
the "10 random trials" requirement; the instructions page carries the worked
explanation instead. (Open to adding a 1–2 trial unscored practice if the
reviewer prefers.)

## Implementation

### PsyNet constructs

- **Trial maker:** `StaticTrialMaker` with one `StaticNode` per stimulus pair.
  - `expected_trials_per_participant = 10`, `max_trials_per_participant = 10`,
    `allow_repeated_nodes = False`, `balance_across_nodes = True`,
    `recruit_mode = "n_participants"`.
  - `target_n_participants` / `target_trials_per_node` chosen so every pair is
    covered across the cohort.
- **Trial:** `StaticTrial` subclass (`SimilarityTrial`). `definition` holds the
  pair; `show_trial` builds the page; `answer` stores `{"rating", "rt_msec",
  ...}`.
- **Page:** `ModularPage` combining:
  - `GraphicPrompt` (PsyNet Native Graphics) with two `Frame`s:
    - Frame 1: fixation cross (`Path`), `duration ≈ 0.8`.
    - Frame 2: the two `Circle` objects, `duration = None` (persists), with
      `activate_control_response = True` and `activate_control_submit = True`.
    - `prevent_control_response = True` on the prompt so the response is locked
      until the stimulus frame, anchoring reaction time to true stimulus onset.
  - A custom `SimilarityRatingControl(KeyboardPushButtonControl)` with
    `choices = ["1".."5"]`, `keys = ["Digit1".."Digit5"]`, arranged horizontally
    and centered under the stimuli.

### Reaction time (no bespoke timing JS)

Reaction time is derived entirely from PsyNet's native **event log** rather than
custom timing JavaScript, per the psychophysics skill. The control overrides
`format_answer` to read `metadata["event_log"]` and compute
`localTime(pushButtonClicked) − localTime(responseEnable)` in milliseconds.
Because `responseEnable` fires at the stimulus frame (via
`activate_control_response`), this equals stimulus-onset-to-response time.

### Simulation hooks

`SimilarityRatingControl.get_bot_response` returns
`BotResponse(raw_answer=<rating>, metadata={"event_log": [...]})` (no `answer`),
so simulated responses flow through the same `format_answer` path and yield real
`rt_msec` values. The simulated rating is drawn from a distribution centered on
hue similarity (closer hues → higher rating; identical pairs → 5), and the
synthetic event log places the keypress a difficulty-dependent interval after
`responseEnable`, so both the rating and RT heatmaps are non-trivial.

### Stimulus generation

`make_stimuli.py` writes `stimuli.json`: a list of `{id, label, hue, hex, size}`
records. `experiment.py` loads the manifest and builds the node list as all
`i <= j` pairs. Adding size variation later means emitting more rows from
`make_stimuli.py`; no experiment-code change is required.

### Layout / timeline

`Timeline`: instructions `InfoPage` → `StaticTrialMaker` (10 trials) → final
`InfoPage`. `get_basic_data` exposes a flat per-trial table (pair ids, hues,
rating, rt) for convenient export/analysis.

### Analysis

Canonical notebook `evidence/analyses/analysis.ipynb` reads the exported trial
CSV directly and produces:

- a **6×6 mean similarity-rating heatmap** over the color pairs (symmetric),
- a **6×6 mean reaction-time heatmap** over the color pairs,
- supporting summary tables (trials per pair, rating distribution) and a short
  interpretation (monotonic similarity vs. hue distance; identical-pair anchor).

### Dependencies

PsyNet only (Native Graphics, static trials, keyboard push-button control). No
external services, no audio, no network calls. `requirements.txt` pins PsyNet to
the checkout commit `d39dab1133b41d8414cf74b91ca153ab796db5e8`;
`constraints.txt` generated with `dallinger constraints generate`.
