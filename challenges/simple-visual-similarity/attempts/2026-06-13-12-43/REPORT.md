## Overview

This attempt implements the **Simple Visual Similarity** experiment as a PsyNet
study. On each trial a participant sees a fixation cross followed by a pair of
colored circles presented simultaneously, and rates how similar they look on a
5-point Likert scale (`1 = Completely Dissimilar` â€¦ `5 = Completely Similar`).
The exact stimulus pair, the rating, and the reaction time are recorded for every
trial. Each participant completes 10 random trials drawn from the full set of
stimulus pairs.

## Implementation

- **Stimuli** (`code/simple_visual_similarity/stimuli.json`, generated by
  `make_stimuli.py`): 6 colored circles of fixed size, with hues evenly spaced
  around the HSL color wheel (red, yellow, green, cyan, blue, magenta). Each
  stimulus record stores a `size` (radius) field, so a **size dimension can be
  added later** by emitting more rows from `make_stimuli.py` â€” no change to
  `experiment.py` is required.
- **Pairs / design**: one `StaticNode` per unordered stimulus pair including
  identical pairs (`i <= j`), giving 21 pairs. A `StaticTrialMaker` administers
  10 random non-repeating trials per participant
  (`expected_trials_per_participant = max_trials_per_participant = 10`,
  `allow_repeated_nodes = False`, `balance_across_nodes = True`), so across the
  cohort every pair is presented many times.
- **Trial display**: a `ModularPage` with a PsyNet Native Graphics
  `GraphicPrompt` containing two frames â€” a fixation-cross frame
  (`Path`, 0.8 s) and a stimulus frame with the two `Circle` objects. The prompt
  uses `prevent_control_response`/`prevent_control_submit`, and the stimulus
  frame uses `activate_control_response`/`activate_control_submit`, so the
  response interface unlocks exactly at stimulus onset.
- **Response**: a `KeyboardPushButtonControl` subclass
  (`SimilarityRatingControl`) with five buttons labeled `1`â€“`5`; participants
  answer by pressing the number keys `1`â€“`5` (the buttons are also clickable).
- **Reaction time**: derived entirely from PsyNet's native event log â€” the
  control's `format_answer` computes
  `localTime(pushButtonClicked) âˆ’ localTime(responseEnable)` in milliseconds.
  Because `responseEnable` fires at the stimulus frame, this is true
  stimulus-onset-to-response time. No bespoke timing JavaScript is used. Each
  trial's answer is stored as `{"rating", "rt_msec", "rating_label"}`.
- **Simulation hooks**: `SimilarityRatingControl.get_bot_response` returns a
  `BotResponse` with `raw_answer` and a synthetic `event_log` (no pre-formatted
  `answer`), so simulated participants flow through the same `format_answer`
  path and yield real `rt_msec` values. Simulated ratings are centered on hue
  similarity and simulated reaction times peak for intermediate (ambiguous)
  pairs, producing non-trivial heatmaps.

## Analysis

`evidence/analyses/analysis.ipynb` reads the exported CSV directly from
`evidence/simulated_data.zip`, parses the JSON stimulus columns, and produces:

- a **mean similarity-rating heatmap** over the 6Ă—6 color pairs,
- a **mean reaction-time heatmap** over the 6Ă—6 color pairs,
- mean rating and mean reaction time as a function of hue distance.

## Findings (simulated data)

- All 21 stimulus pairs (including the 6 identical pairs) were presented multiple
  times, so every cell of the similarity matrix is estimated.
- Mean similarity ratings decrease monotonically with hue distance; identical
  pairs (the matrix diagonal) are rated most similar and opposite hues least
  similar.
- Reaction times are longest for intermediate hue distances (the most ambiguous
  pairs) in this simulated dataset.

These patterns reflect the bot generative model rather than human perception.
Re-running the same notebook on a real `psynet export` archive would reveal the
empirical structure.
