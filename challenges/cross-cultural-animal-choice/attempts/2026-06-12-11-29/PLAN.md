# Plan

## Science

This experiment will measure simple animal preference judgments across participant language or locale groups. The scientific goal is descriptive rather than confirmatory: estimate how often participants choose cat, dog, or bird for two culturally interpretable prompts, then compare those choice proportions by locale or language setting. The design intentionally minimizes participant burden by limiting the task to exactly two choice trials.

## Methods

Participants will complete a short PsyNet experiment consisting of a welcome page, brief instructions, two forced-choice trials, and a thank-you page. The three response options will be cat, dog, and bird. Each option will be shown with a short label and a simple visual symbol. To avoid relying on external assets, I plan to use Unicode animal symbols where they render reliably; if testing shows rendering problems, I will replace them with small locally bundled SVG assets.

Each participant will see the same two prompts:

1. "Which animal would you most like to have as a companion?"
2. "Which animal do you think is most respected in your community?"

The order of the two prompts will remain fixed so that every participant answers the companion question before the community-respect question. Within each trial, the three animal options will be independently randomized, and the recorded data will include the prompt key, prompt text, randomized animal order, chosen animal key, response position, reaction time, participant locale, and browser language when available. The experiment will not ask for names, addresses, ethnicity, nationality, or other identifying cultural information; locale and browser language will be used as the safe cultural-context metadata.

The primary analysis will compute choice proportions by prompt and locale or language group. A lightweight analysis notebook will read simulated or exported trial data, tabulate choices, and produce bar plots for the two prompts.

## Implementation

The experiment will live under `code/cross_cultural_animal_choice/` so it does not collide with Python's standard-library `code` module. I will use a self-contained PsyNet experiment with standard support files copied or adapted from PsyNet demos where appropriate.

The PsyNet implementation will use:

- a subclass of `psynet.experiment.Experiment`;
- a `Timeline` containing a welcome page, instructions page, two modular choice pages, and a thank-you page;
- `ModularPage` with `PushButtonControl` for the three animal choices, because the challenge asks for a simple simultaneous three-option response interface and optional keyboard shortcuts;
- deterministic prompt definitions in Python, with per-participant trial pages generated by timeline logic or a small helper that randomizes animal order at page construction time;
- custom page metadata or response processing to save prompt key, animal order, selected animal, selected position, reaction time, locale, and browser language.

Because the challenge is explicitly cross-cultural, all participant-facing text will be marked with PsyNet's translation helper:

- import `get_translator` and define `_ = get_translator()` at module scope;
- wrap literal instructions, prompts, button labels, and thank-you text in `_()`;
- avoid f-strings or string concatenation inside translation calls;
- set source-locale configuration in `config.txt` or experiment `config`, and leave target locale generation for later localization unless specific target languages are requested.

After implementation, I will verify translation readiness by running a safe gettext/PsyNet extraction path and checking that expected participant-facing strings appear in `locales/experiment.pot`. I will not configure real translation API credentials.

Planned validation and evidence:

- run `python experiment.py` from the experiment directory;
- run `psynet test local`;
- run `psynet simulate` to generate representative simulated data;
- run the analysis notebook or script on simulated/exported data;
- run `psynet performance-test local --n-bots 40 --duration-minutes 5 --time-factor 1.0 --json-output <absolute evidence path>`;
- collect participant-flow screenshots and `evidence/participant.mp4` using the participant-video workflow;
- export experiment data to `evidence/data.zip`;
- save any PsyNet monitor snapshot required by the challenge evidence checklist.

## Review questions

- Should the two prompts remain fixed in this order, or should prompt order be randomized while still limiting each participant to exactly two trials?
- Are cat, dog, and bird acceptable as labels plus simple symbols, or should this attempt use bundled SVG images from the start?
- Should the experiment declare any specific target locales now, or remain source-locale-ready until a later localization phase?
