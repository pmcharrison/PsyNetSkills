# Plan

## Science

This experiment will implement a compact visual psychophysics battery that measures three related behaviors over the same 30-color stimulus space: categorical same-different judgments, explicit similarity ratings, and multi-item identification/generalization. The shared color-circle stimulus set supports direct comparison of empirical matrices across tasks.

## Methods

The battery will contain three experimental blocks. Block order will be fixed as discrimination, similarity, then multi-item identification so participants first learn the simple stimulus-response mappings before the more complex array task. Each block will contain 10 trials per participant, sampled from a larger generated pool of trial definitions and balanced as far as practical across participants by PsyNet's node allocation.

The stimulus set will comprise 30 colors evenly spaced on the HSV hue circle. Each color will have a stable integer id, hue value, and hex representation. Stimuli will be rendered as filled circles on a white field with no extra frame or border. Circle size will be fixed and large enough to approximate the requested quarter-display visual salience while leaving clear separation between paired or arrayed items. The stimulus-definition helpers will keep color and size in structured dictionaries so later visual dimensions, such as size, can be added without changing the trial schema.

In the discrimination block, each trial will begin with a 500 ms fixation cross, followed by two colored circles shown simultaneously for 1000 ms. The display will then be blank for 500 ms before a two-alternative response appears outside the visual field. Trials will include both same and different pairs, with same trials using identical color ids and different trials using distinct color ids sampled from the hue circle. Participants will respond "Same" or "Different" using keyboard shortcuts and mouse-clickable buttons. The recorded answer will include the pair ids, condition, response, correctness, and reaction time measured from response onset.

In the similarity block, each trial will begin with a 500 ms fixation cross, followed by two colored circles shown simultaneously. The pair display will remain visible while participants rate similarity from 1 = Completely Dissimilar to 5 = Completely Similar. The full node pool will include all unordered color pairs with replacement so diagonal self-pairs can appear, while each participant receives 10 random balanced trials. The recorded answer will include the pair ids, rating, rating normalized by 5, and reaction time measured from stimulus onset.

In the multi-item identification block, each trial will begin with a 500 ms fixation cross, followed by N numbered colored circles arranged evenly around fixation, with N sampled across 3, 4, and 5. The array will be shown for a fixed 1000 ms duration and then replaced by a blank display for at least 500 ms. A single probe circle will then appear. On present-probe trials, the probe color will match one array item and the correct response will be the matching item number. On novel-probe trials, the probe color will be outside the displayed set and the response will be interpreted as a similarity-based generalization choice. Participants will choose an item number using digit keys or mouse-clickable buttons. The recorded answer will include the full item set, item numbers, item positions, probe id, probe condition, selected item number, selected color id, accuracy where defined, and reaction time measured from probe onset.

The participant-facing flow will include concise instructions before each block and a final completion page. Task reminders will appear outside the visual stimulus area, and the visual frames will contain only the specified fixation, circles, item numbers for block 3, and probe. Neutral CSS will be used for buttons and progress indicators to avoid introducing color-biased interface chrome.

## Implementation

The experiment will live under `code/visual_psychophysics_battery/` to avoid importing a package named `code`. It will follow the local PsyNet static and graphics demo patterns: `StaticNode` objects will hold trial definitions, a `StaticTrial` subclass will render each trial, and a `StaticTrialMaker` subclass will administer the three PsyNet blocks. Nodes will use the `block` field with values `discrimination`, `similarity`, and `identification`; the trial maker will keep a fixed block order and set `max_trials_per_block=10`.

Trial rendering will use `ModularPage` with `GraphicPrompt` frames. Fixation, stimulus, blank, and response/probe phases will be implemented with timed `Frame` objects. Control responses will be disabled during non-response frames with `prevent_control_response=True` and `prevent_control_submit=True`, then enabled with `activate_control_response=True` and `activate_control_submit=True` at the frame that defines the reaction-time origin for that block.

Response controls will subclass `KeyboardPushButtonControl` so every task supports both keyboard shortcuts and mouse clicks. The control `format_answer` methods will read PsyNet's native event log and compute reaction time from `responseEnable` to the relevant `pushButtonClicked` event. For bots, `get_bot_response` will return a `BotResponse` with synthetic event-log timestamps so simulations exercise the same formatting path and produce real `rt_msec` values.

Simulation behavior will use the requested probabilistic hue-distance model. For a color pair or probe/choice pair, distance will be the circular hue distance and choice weights will be proportional to `exp(-d(x, y))`. Discrimination bots will be more likely to answer "same" for low distances and especially exact matches. Similarity bots will map distance-based similarity to a 1-5 rating with noise. Identification bots will sample choices from the array using the same exponential distance model, yielding high accuracy for present probes and graded generalization for novel probes.

The analysis notebook at `evidence/analyses/analysis.ipynb` will read the simulated export CSV files directly. It will summarize accuracy or response distributions by block and condition, reaction times by block and condition, and construct three 30 x 30 empirical matrices: the normalized count of "same" responses for block 1, the mean similarity rating divided by 5 for block 2, and the normalized probe-to-selected-color counts for block 3. The three matrices will be displayed side by side with `plt.matshow` and a shared color bar.

Validation after implementation will follow the experiment evidence checklist: `python experiment.py`, `psynet test local`, `psynet simulate` with enough bots to populate all analysis paths, `psynet performance-test local` with JSON output under `evidence/performance.json`, a Playwright-driven participant recording saved as `evidence/participant.mp4`, targeted screenshots under `evidence/screenshots/`, a monitor snapshot, the executed analysis notebook, and `REPORT.md`. Any blocked check will be recorded in `EVALUATION.md` rather than presented as passing.
