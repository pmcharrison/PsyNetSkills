# Evaluation

## Summary

The implementation is generally good and satisfies the public challenge
requirements, with a clear sequence-recall flow, saved analyzable responses, and
strong local evidence. The main shortcomings are experiment-design choices that
would matter in a real memory study: missing practice/training, allowing replay,
using browser synthesis instead of generated audio files, and hardcoding
stimuli directly in `experiment.py`.

## Strengths

- Implements the required four-trial tone memory task with three tone labels.
- Uses a PsyNet trial abstraction rather than duplicating trial pages manually.
- Saves target sequences, participant responses, accuracy, and position-level
  scoring in exported data.
- Provides strong evidence, including an audio-enabled participant video,
  monitor snapshot, data export, and automated PsyNet tests.

## Weaknesses

- Lacks a training/practice phase where participants learn the task and try
  practice trials before scored trials.
- Includes a replay button, which gives a large advantage in a memory task.
- Uses `JSSynth`; generated audio files from Python would provide more control
  and easier replication.
- Hardcodes stimuli in `experiment.py` rather than generating or loading a
  stimulus definition file such as JSON.
