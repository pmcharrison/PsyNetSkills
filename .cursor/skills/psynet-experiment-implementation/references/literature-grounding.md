# Literature grounding

Use this procedure when the plan needs a canonical reference (replications and
adaptations) or when grounding a new science question in prior work. The same
procedure applies when verifying reference materials for a challenge via the
`create-challenge` skill.

## Procedure

1. **Confirm consent to search.** In interactive mode, ask before searching
   online whether the user wants to provide materials themselves, wants you to
   search, or wants to treat the idea as novel with no canonical reference. Do
   not search before the user has agreed that search is wanted.
2. **Build queries** from the research question and constructs: the construct
   itself, the paradigm family, and likely effect-size sources.
3. **Verify every source against reality.** Fetch the DOI, arXiv page, or
   publisher page before citing. Never invent citations, DOIs, URLs,
   summaries, or bibliographic details. If a claimed paper cannot be located,
   say so rather than approximating.
4. **Store and record.** Put verified materials in the experiment or
   challenge folder under `references/literature/`:
   - If the license permits redistribution, store a copy (PDF or HTML).
   - Otherwise store a short notes file with the citation, URL, and the
     specific facts extracted, and record only citation plus URL in PLAN.md.
5. **Transcribe the procedure.** For replications, copy exact instruction
   wording, question text, response scales, timing values, and screening
   criteria from the source into the Science and Method sections, noting what
   was transcribed directly.
6. **Record expected effect sizes** (or a justified range) when prior work
   reports them; they feed the sample-size rationale. If no strong priors
   exist, write that explicitly rather than inventing a number.

## Outputs

- Verified files or notes under `references/literature/` of the experiment or
  challenge folder (development materials such as mock experiments, manifests,
  and sketches belong under `references/experiment/` instead).
- A canonical-reference line at the top of PLAN.md.
- Decision-log entries for any source that was considered and rejected.
